# ADR-003: VS Code extension and backend communicate through an explicit, versioned contract

**Date:** 2026-04-10T19:50:15.112Z  
**Status:** accepted  
**Decided by:** collaborative  
**Affected entities:** dreamgraph_extensions_vscode_src_types, feature_cognitive_engine  

## Context

The extension depends on backend/daemon behavior, but there is no ADR defining the trust boundary, failure handling expectations, or compatibility contract between the VS Code client and DreamGraph services.

**Constraints:**
- The extension must remain usable when backend services are degraded or unavailable
- Webview and extension-host state can diverge if contracts are implicit
- The project may evolve backend APIs and extension UX independently

## Decision

Treat the VS Code extension ↔ backend interface as an explicit contract. Requests and responses should be versionable, capability-aware where needed, and designed for graceful degradation when the backend is unavailable.

### Alternatives Considered

- **Rely on informal coupling between current extension code and server implementation** — Rejected: This increases breakage risk and makes compatibility failures hard to diagnose.
- **Push all state into the webview and use the extension host as a thin proxy** — Rejected: This weakens host-side control over persistence, recovery, and instance-aware orchestration.

## Consequences

- Backend changes become easier to reason about and stage
- The extension can implement clearer degraded/offline UX
- State ownership remains concentrated in the extension host rather than fragmented in the webview

**Risks accepted:**
- Versioning introduces some protocol maintenance overhead
- Capability mismatches need to be surfaced intentionally

## Guard Rails

> ⚠️ Do NOT introduce breaking extension/backend message changes without a versioning or compatibility story.

> ⚠️ Do NOT make the webview the sole source of truth for persistent chat or instance state.

> ⚠️ Do NOT assume backend availability in user-facing extension flows without a degraded-mode response.
