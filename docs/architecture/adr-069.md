# ADR-069: Data Storage Strategy

**Date:** 2026-04-25T11:31:55.084Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dreamgraph_src_instance, dreamgraph_src_config  

## Context

Need to define where different types of data will be stored.

**Constraints:**
- Transactional data must be easily queryable
- Configuration data should be human-readable

## Decision

All transactional data uses a database, configuration uses JSON files.

### Alternatives Considered

- **Use a single database for all data** — Rejected: Configuration data is not suitable for database storage.
- **Store all data in flat files** — Rejected: Querying transactional data would be inefficient.

## Consequences

- Improved data retrieval performance for transactional data
- Easier configuration management

**Risks accepted:**
- Potential for configuration file corruption

## Guard Rails

> ⚠️ Do NOT change the storage strategy without reviewing this ADR
