# ADR-017: Integration Pattern for External Services

**Date:** 2026-04-18T23:11:34.801Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** cognitive_engine, api_surface  

## Context

To enable seamless communication with external services while maintaining system integrity.

**Constraints:**
- All external integrations must use webhooks for asynchronous communication

## Decision

Implement webhook-based integrations for external services.

### Alternatives Considered

- **Polling for external service updates** — Rejected: Inefficient and increases latency.
- **Direct API calls** — Rejected: Tight coupling with external services.

## Consequences

- Reduced latency and improved responsiveness to external events.

**Risks accepted:**
- Webhook failures could lead to missed events.

## Guard Rails

> ⚠️ Do NOT change integration patterns without reviewing this ADR.
