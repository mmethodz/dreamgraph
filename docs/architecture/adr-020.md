# ADR-020: Testing Framework Selection

**Date:** 2026-04-18T23:11:34.850Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** test_framework, test_suite  

## Context

To ensure reliable testing of the application components.

**Constraints:**
- All tests must be automated and easily runnable

## Decision

Use Vitest as the testing framework for the application.

### Alternatives Considered

- **Use Mocha** — Rejected: Less integrated with the current stack.
- **Manual testing only** — Rejected: Not scalable and prone to human error.

## Consequences

- Increased reliability and faster feedback loops during development.

**Risks accepted:**
- Learning curve for developers unfamiliar with Vitest.

## Guard Rails

> ⚠️ Do NOT change the testing framework without reviewing this ADR.
