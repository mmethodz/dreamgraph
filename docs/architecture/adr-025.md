# ADR-025: Use of JSON for Configuration Management

**Date:** 2026-04-18T23:11:40.695Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dreamgraph_src_config  

## Context

To provide a simple and human-readable format for configuration settings.

**Constraints:**
- Configuration files must be in JSON format
- Must support version control

## Decision

Configuration settings are stored in JSON files.

### Alternatives Considered

- **YAML** — Rejected: Less common in the existing ecosystem.
- **XML** — Rejected: Verbosity and complexity.

## Consequences

- Easier to read and modify configuration settings.

**Risks accepted:**
- Potential for syntax errors in JSON files.

## Guard Rails

> ⚠️ Do NOT change configuration format without reviewing this ADR
