# ADR-029: Isolation of Invoice Processing from User Management

**Date:** 2026-04-18T23:11:40.765Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** feature, feature  

## Context

To maintain clear domain boundaries and reduce complexity.

**Constraints:**
- Invoice processing must be separate from user management
- Must ensure data integrity between domains

## Decision

Invoice processing is isolated from user management.

### Alternatives Considered

- **Combined processing** — Rejected: Increased complexity and risk of errors.
- **Shared data models** — Rejected: Tight coupling between domains.

## Consequences

- Easier maintenance and scalability.

**Risks accepted:**
- Data synchronization issues between domains.
