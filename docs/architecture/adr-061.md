# ADR-061: Use of JSON for API Contracts

**Date:** 2026-04-25T11:03:26.964Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** api_contract  

## Context

Need a clear and structured format for API contracts.

**Constraints:**
- API contracts must be defined in JSON format.

## Decision

JSON format for API contracts.

### Alternatives Considered

- **XML** — Rejected: More verbose and less readable.
- **YAML** — Rejected: Less common for API contracts.

## Consequences

- Easier to read and maintain API contracts.

**Risks accepted:**
- JSON schema validation errors.

## Guard Rails

> ⚠️ Do NOT change API contract format without reviewing this ADR.
