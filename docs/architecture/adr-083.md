# ADR-083: Normalize VS Code extension UI registry IDs to semantic generic identifiers with platform-specific implementations

**Date:** 2026-04-26T23:50:07.599Z  
**Status:** accepted  
**Decided by:** collaborative  
**Affected entities:** dashboard_view, changed_files_view, graph_signal_indicator, context_inspector_output, dreamgraph_chat_panel, dreamgraph_tool_progress_view, dreamgraph_recommended_actions_bar, dreamgraph_status_bar_indicator  

## Context

The UI registry for the VS Code extension uses a mix of generic semantic IDs and DreamGraph-prefixed IDs for equivalent extension-facing UI surfaces, which makes discovery and cross-platform portability inconsistent.

**Constraints:**
- Existing UI registry entries already use semantic platform-independent modeling.
- UI registry must preserve purpose, contracts, and platform implementations rather than encode platform identity in IDs.
- No source-code behavior changes are required for naming normalization.

## Decision

Use generic semantic UI element IDs as the canonical registry naming convention, and represent the VS Code-specific realization through implementation.platform='vscode' plus source-file anchors rather than DreamGraph-prefixed element IDs.

### Alternatives Considered

- **Keep mixed generic and dreamgraph-prefixed IDs** — Rejected: Leads to inconsistent lookup, unclear canonical identity, and duplicate concepts for the same UI surfaces.
- **Standardize everything on dreamgraph-prefixed IDs** — Rejected: Bakes product/platform naming into semantic element identity and weakens portability across platforms.

## Consequences

- Registry entries will be easier to search and reason about semantically.
- Cross-platform UI portability improves because IDs represent concepts rather than implementations.
- Existing extension-prefixed entries should be treated as transitional aliases or candidates for consolidation.

**Risks accepted:**
- Some existing tooling or prompts may still reference prefixed IDs until the registry is fully consolidated.
- Duplicate entries may temporarily coexist during migration.

## Guard Rails

> ⚠️ Do not introduce new extension UI registry IDs that encode the implementation platform when a semantic generic identifier can represent the same concept.

> ⚠️ Preserve VS Code-specific details under implementations metadata rather than in canonical UI element IDs.
