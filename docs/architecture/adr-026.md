# ADR-026: Supabase for Transactional Data Storage

**Date:** 2026-04-18T23:11:40.713Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** data_model_feature, data_model_workflow  

## Context

To provide a reliable and scalable solution for managing transactional data.

**Constraints:**
- All transactional data must be stored in Supabase
- Must support real-time updates

## Decision

Supabase is used for all transactional data storage.

### Alternatives Considered

- **PostgreSQL** — Rejected: Requires more manual setup and management.
- **MongoDB** — Rejected: Not suitable for relational data.

## Consequences

- Improved data integrity and scalability.

**Risks accepted:**
- Dependency on Supabase service availability.

## Guard Rails

> ⚠️ Do NOT switch database providers without reviewing this ADR
