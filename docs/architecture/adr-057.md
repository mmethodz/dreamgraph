# ADR-057: Webhook-Based Async Integration with External Services

**Date:** 2026-04-25T11:03:26.886Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** feature_tool_registry  

## Context

Need to integrate with external services without blocking the main application flow.

**Constraints:**
- All external integrations must use webhooks.

## Decision

Webhook-based integration for external services.

### Alternatives Considered

- **Polling** — Rejected: Less efficient and increases load.
- **Direct API Calls** — Rejected: Blocks application flow.

## Consequences

- Improved responsiveness and scalability.

**Risks accepted:**
- Webhook delivery failures may lead to data inconsistencies.

## Guard Rails

> ⚠️ Do NOT change integration pattern without reviewing this ADR.
