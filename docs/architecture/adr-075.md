# ADR-075: Data Storage Choices for Configuration

**Date:** 2026-04-25T11:55:43.065Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** config, configuration  

## Context

Configuration data needs to be stored in a format that is easy to read and modify.

**Constraints:**
- Configuration must be easily editable by developers.
- Configuration should be lightweight.

## Decision

All configuration data uses JSON files.

### Alternatives Considered

- **YAML** — Rejected: Less common in the team and could introduce parsing complexity.

## Consequences

- Configuration changes can be made quickly and easily.

**Risks accepted:**
- Potential for malformed JSON leading to runtime errors.

## Guard Rails

> ⚠️ Do NOT change the configuration format without reviewing this ADR.
