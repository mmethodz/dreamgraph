# ADR-074: Domain Separation

**Date:** 2026-04-25T11:31:55.191Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dreamgraph_src_cognitive, dreamgraph_src_instance  

## Context

Need to ensure clear boundaries between different functional areas.

**Constraints:**
- Each domain must be independently deployable
- Minimize coupling between domains

## Decision

Invoice processing is isolated from user management.

### Alternatives Considered

- **Combine all domains into a single service** — Rejected: Increases complexity and reduces maintainability.
- **Use a monolithic architecture** — Rejected: Limits scalability.

## Consequences

- Easier to manage and scale individual domains

**Risks accepted:**
- Potential duplication of logic across domains

## Guard Rails

> ⚠️ Do NOT change domain boundaries without reviewing this ADR
