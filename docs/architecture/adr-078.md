# ADR-078: Architectural Patterns for Server-Side Rendering

**Date:** 2026-04-25T11:55:43.126Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dashboard_server  

## Context

The application needs to render pages dynamically based on user interactions.

**Constraints:**
- Rendering must be efficient and responsive.

## Decision

Server-side rendering via Next.js for all pages.

### Alternatives Considered

- **Static site generation** — Rejected: Not suitable for dynamic content.

## Consequences

- Improved performance and SEO.

**Risks accepted:**
- Increased complexity in managing server-side code.

## Guard Rails

> ⚠️ Do NOT change the rendering strategy without reviewing this ADR.
