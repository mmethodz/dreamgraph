# ADR-037: Integration Pattern for External Services

**Date:** 2026-04-18T23:12:58.731Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** api_surface  

## Context

To efficiently integrate with external services while ensuring reliability.

**Constraints:**
- Asynchronous communication is preferred
- Webhooks must be used for event-driven interactions

## Decision

Implement webhook-based asynchronous integration for external services.

### Alternatives Considered

- **Synchronous API calls** — Rejected: Can lead to blocking operations and reduced performance
- **Polling for updates** — Rejected: Inefficient and can lead to unnecessary load on services

## Consequences

- Improved responsiveness and scalability
- Reduced load on external services

**Risks accepted:**
- Webhook delivery failures may lead to missed events
- Increased complexity in handling asynchronous workflows

## Guard Rails

> ⚠️ Do NOT change the integration pattern without reviewing this ADR
