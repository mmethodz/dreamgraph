# ADR-072: Integration with External Services

**Date:** 2026-04-25T11:31:55.149Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dreamgraph_src_cognitive  

## Context

Need a reliable way to integrate with external services without blocking operations.

**Constraints:**
- Integration must not block user interactions
- Must handle failures gracefully

## Decision

Webhook-based async integration with external services.

### Alternatives Considered

- **Polling for updates** — Rejected: Inefficient and increases load.
- **Synchronous API calls** — Rejected: Could block user interactions.

## Consequences

- Improved responsiveness of the application
- Better user experience

**Risks accepted:**
- Webhook delivery failures could lead to missed updates

## Guard Rails

> ⚠️ Do NOT change the integration pattern without reviewing this ADR
