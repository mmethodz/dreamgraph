# ADR-028: Webhook-based Integration with External Services

**Date:** 2026-04-18T23:11:40.748Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** feature_tool_registry  

## Context

To allow asynchronous communication with external services.

**Constraints:**
- All external integrations must use webhooks
- Must handle retries for failed requests

## Decision

Webhook-based integration is implemented for external services.

### Alternatives Considered

- **Polling** — Rejected: Inefficient and increases latency.
- **Direct API calls** — Rejected: Tight coupling with external services.

## Consequences

- Improved responsiveness and reduced load.

**Risks accepted:**
- Webhook delivery failures may lead to data inconsistency.

## Guard Rails

> ⚠️ Do NOT change integration method without reviewing this ADR
