# ADR-035: Data Storage Strategy

**Date:** 2026-04-18T23:12:58.694Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** data_model, config  

## Context

To define a clear strategy for data storage that optimizes performance and maintainability.

**Constraints:**
- Transactional data must be stored in memory for fast access
- Configuration data must be stored in JSON files

## Decision

Use in-memory storage for transactional data and JSON files for configuration.

### Alternatives Considered

- **All data in a relational database** — Rejected: Slower access for frequently used transactional data
- **No structured storage** — Rejected: Leads to data inconsistency and management issues

## Consequences

- Faster data access for transactional operations
- Simplified configuration management

**Risks accepted:**
- Memory limitations may affect scalability
- JSON files may become cumbersome for complex configurations

## Guard Rails

> ⚠️ Do NOT change the storage strategy without reviewing this ADR
