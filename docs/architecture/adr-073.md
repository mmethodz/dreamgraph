# ADR-073: Architectural Pattern for Rendering

**Date:** 2026-04-25T11:31:55.169Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dreamgraph_extensions_vscode  

## Context

Need to determine how pages will be rendered for optimal performance.

**Constraints:**
- Must support server-side rendering for SEO
- Should minimize client-side load

## Decision

Server-side rendering via Next.js for all pages.

### Alternatives Considered

- **Client-side rendering only** — Rejected: SEO would suffer.
- **Static site generation** — Rejected: Not suitable for dynamic content.

## Consequences

- Improved SEO
- Faster initial page loads

**Risks accepted:**
- Increased server load during peak traffic

## Guard Rails

> ⚠️ Do NOT change the rendering strategy without reviewing this ADR
