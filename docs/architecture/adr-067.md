# ADR-067: Server-side Rendering for Webviews

**Date:** 2026-04-25T11:08:30.188Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dreamgraph_extensions_vscode_src_webview  

## Context

Need for fast initial load times and SEO optimization.

**Constraints:**
- All webviews must be rendered on the server
- Must support dynamic content

## Decision

Server-side rendering using Next.js

### Alternatives Considered

- **Client-side rendering** — Rejected: Slower initial load times and poor SEO
- **Static site generation** — Rejected: Not suitable for dynamic content

## Consequences

- Faster load times
- Better SEO performance

**Risks accepted:**
- Increased server load during peak times

## Guard Rails

> ⚠️ Do NOT change rendering strategy without reviewing this ADR
