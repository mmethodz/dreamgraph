# ADR-044: Architect chat must render structured execution artifacts eagerly and normalize multimodal attachments per provider

**Date:** 2026-04-19T21:39:15.402Z  
**Status:** accepted  
**Decided by:** collaborative  
**Affected entities:** feature_architect_structured_webview, feature_provider_multimodal_attachment_translation, feature_agentic_tool_execution_loop, feature_architect_tool_call_translation, workflow_architect_agentic_tool_execution  

## Context

The architect chat panel had evolved from a stream-only text surface into a structured execution surface, but rich artifacts such as verdict frames, expandable tool traces, pills/chips, and interactive references were not guaranteed to render eagerly. In parallel, multimodal clipboard/image attachments crossed provider boundaries with incompatible payload shapes, creating failures when Anthropic-style image content was sent through the OpenAI path.

**Constraints:**
- ADR-006 requires architect chat to use structured tool-calling rather than stream-only narration.
- ADR-007 requires preserving the end-to-end agentic tool execution loop in the VS Code chat panel.
- The chat webview must remain the canonical presentation layer for execution state, verification state, and tool-use context.
- Provider APIs differ in multimodal image content contracts, so translation must be normalized at the provider bridge rather than leaking provider-specific shapes into the UI layer.
- Interactive file/entity references and rich execution artifacts must remain compatible with the structured webview model.

## Decision

The architect chat panel is defined as a structured execution surface. Rich execution artifacts (including verdict/status framing, expandable tool traces, pills/chips, and interactive entity/file references) must render eagerly at message creation time rather than depending on later repaint/reflow side effects. Multimodal clipboard/image attachments must be normalized in the provider translation layer using provider-specific output shapes while preserving a provider-agnostic UI attachment model.

### Alternatives Considered

- **Keep stream-first rendering and progressively enhance rich artifacts later** — Rejected: This weakens ADR-006/ADR-007 goals by making the structured execution loop visually unreliable and delaying critical execution context.
- **Handle provider image shape differences directly in the webview or shared UI model** — Rejected: This leaks provider-specific contracts into presentation concerns and makes multimodal behavior harder to reason about and maintain across providers.
- **Allow lazy/conditional rendering of traces and verdict frames based on user interaction or reflow timing** — Rejected: This caused inconsistent visibility of execution artifacts and reduced trust in the structured assistant interface.

## Consequences

- The chat panel consistently presents execution state as structured UI rather than delayed text decoration.
- Rich assistant artifacts become visible immediately, improving usability and trust during agentic tool execution.
- Provider-specific multimodal differences are isolated to the translation layer, simplifying the UI contract.
- Future changes to architect chat can be evaluated against a clear boundary between presentation semantics and provider adaptation.

**Risks accepted:**
- Eager rendering may expose latent performance issues in the webview if structured artifacts become too heavy.
- Provider translation logic may require maintenance as upstream APIs evolve.
- Developers may accidentally reintroduce provider-specific assumptions into UI-layer code if guard rails are not followed.

## Guard Rails

> ⚠️ Do NOT revert the architect chat panel to a stream-only narration model without revisiting this ADR.

> ⚠️ Do NOT make verdict frames, structured traces, pills/chips, or interactive references depend on deferred repaint/reflow side effects for initial visibility.

> ⚠️ Do NOT encode Anthropic/OpenAI-specific multimodal payload shapes directly into the webview UI contract; normalize them in the provider translation layer.

> ⚠️ Do NOT remove or bypass the structured execution artifact presentation in ways that obscure the end-to-end agentic tool execution loop.
