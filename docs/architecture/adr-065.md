# ADR-065: Separation of Concerns in Domain Models

**Date:** 2026-04-25T11:08:30.148Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dreamgraph_extensions_vscode_src, dreamgraph_src_cognitive, dreamgraph_src_instance  

## Context

Need to maintain clear boundaries between different domains to avoid tight coupling.

**Constraints:**
- Each domain must be independently deployable
- No direct dependencies between domains

## Decision

Isolated domain models for each feature

### Alternatives Considered

- **Monolithic domain model** — Rejected: Increases complexity and coupling
- **Shared domain model** — Rejected: Leads to tight coupling and harder maintenance

## Consequences

- Easier maintenance and scalability
- Clearer boundaries for feature development

**Risks accepted:**
- Potential for duplicated logic across domains

## Guard Rails

> ⚠️ Do NOT merge domain models without reviewing this ADR
