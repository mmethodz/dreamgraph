# ADR-070: API Authentication Method

**Date:** 2026-04-25T11:31:55.107Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dreamgraph_src_api  

## Context

Need to secure API endpoints to prevent unauthorized access.

**Constraints:**
- All internal routes must be secured
- User credentials must not be exposed

## Decision

API key authentication is mandatory for internal routes.

### Alternatives Considered

- **OAuth 2.0** — Rejected: Overhead for internal services.
- **No authentication** — Rejected: Security risks are too high.

## Consequences

- Increased security for API endpoints

**Risks accepted:**
- API key leakage could compromise security

## Guard Rails

> ⚠️ Do NOT change the authentication method without reviewing this ADR
