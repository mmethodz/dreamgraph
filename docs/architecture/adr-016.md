# ADR-016: Data Storage Strategy

**Date:** 2026-04-18T23:11:34.773Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** data_model, config  

## Context

To ensure efficient and reliable data management across the application.

**Constraints:**
- Transactional data must be stored in a relational database
- Configuration data must be stored in JSON files

## Decision

Use a relational database for transactional data and JSON files for configuration.

### Alternatives Considered

- **Store all data in a NoSQL database** — Rejected: Does not meet transactional consistency requirements.
- **Use XML for configuration** — Rejected: JSON is more lightweight and easier to parse.

## Consequences

- Improved data integrity and easier configuration management.

**Risks accepted:**
- Migration challenges if data storage strategy changes in the future.

## Guard Rails

> ⚠️ Do NOT change data storage formats without reviewing this ADR.
