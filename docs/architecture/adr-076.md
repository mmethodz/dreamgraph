# ADR-076: Data Storage Choices for Instances

**Date:** 2026-04-25T11:55:43.087Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** instance, policy_profile  

## Context

Instance data needs to support transactions and relationships between instances and policies.

**Constraints:**
- Data must support relational queries.
- Data must be persistent.

## Decision

All instance data uses SQLite.

### Alternatives Considered

- **PostgreSQL** — Rejected: Overhead of managing a separate database server.

## Consequences

- Efficient management of instance lifecycles and policies.

**Risks accepted:**
- SQLite may not scale well with high concurrency.

## Guard Rails

> ⚠️ Do NOT change the database technology without reviewing this ADR.
