# ADR-056: Server-Side Rendering with Next.js

**Date:** 2026-04-25T11:03:26.868Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dreamgraph_extensions_vscode_src  

## Context

Need for fast initial page loads and SEO optimization.

**Constraints:**
- All pages must use server-side rendering.

## Decision

Next.js for server-side rendering.

### Alternatives Considered

- **Static Site Generation** — Rejected: Not suitable for dynamic content.
- **Client-Side Rendering** — Rejected: Slower initial load times.

## Consequences

- Improved performance and SEO for the application.

**Risks accepted:**
- Increased server load due to rendering.

## Guard Rails

> ⚠️ Do NOT change rendering strategy without reviewing this ADR.
