# ADR-071: UI Component Reusability

**Date:** 2026-04-25T11:31:55.127Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** dreamgraph_extensions_vscode_src_webview  

## Context

Need to maintain consistency across the UI and reduce duplication.

**Constraints:**
- All data tables must follow a consistent design
- Components should be reusable

## Decision

All data tables use a shared DataGrid component.

### Alternatives Considered

- **Create unique tables for each use case** — Rejected: Increases maintenance burden.
- **Use third-party libraries for tables** — Rejected: Dependency management issues.

## Consequences

- Faster development of new features
- Easier to maintain UI consistency

**Risks accepted:**
- Changes to the DataGrid component could affect all tables

## Guard Rails

> ⚠️ Do NOT modify the DataGrid component without reviewing this ADR
