# ADR-027: Server-Side Rendering with Next.js

**Date:** 2026-04-18T23:11:40.731Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** feature_dashboard_server, feature_vscode_extension  

## Context

To enhance performance and SEO for the web application.

**Constraints:**
- All pages must be rendered server-side
- Must support dynamic content

## Decision

Next.js is used for server-side rendering of all pages.

### Alternatives Considered

- **Client-side rendering** — Rejected: Worse performance and SEO.
- **Static site generation** — Rejected: Inflexible for dynamic content.

## Consequences

- Faster load times and better SEO.

**Risks accepted:**
- Increased server load due to rendering.

## Guard Rails

> ⚠️ Do NOT change rendering strategy without reviewing this ADR
