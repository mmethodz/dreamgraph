# ADR-079: Integration Patterns with External Services

**Date:** 2026-04-25T11:55:43.146Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** cognitive_engine, llm_provider  

## Context

The application needs to communicate with external services efficiently.

**Constraints:**
- Integration must be reliable and maintainable.

## Decision

Webhook-based async integration with external services.

### Alternatives Considered

- **Polling** — Rejected: Less efficient and may lead to unnecessary load.

## Consequences

- Real-time updates from external services.

**Risks accepted:**
- Webhook failures could lead to data inconsistency.

## Guard Rails

> ⚠️ Do NOT change the integration method without reviewing this ADR.
