# ADR-048: Keep rewritten ContextBuilder as default for VS Code context assembly

**Date:** 2026-04-21T01:49:29.527Z  
**Status:** accepted  
**Decided by:** collaborative  
**Affected entities:** extensions-vscode, context-builder, benchmark-context-assembly  

## Context

The project needed a durable architectural decision on whether the rewritten VS Code context assembly approach should remain the default instead of the older broader baseline strategy.

**Constraints:**
- Decision must remain grounded in the completed P01–P10 benchmark artifacts and benchmark conclusion.
- Benchmark artifacts are comparative validation captures, not raw low-level telemetry exports.
- Future regressions must be evaluated against the same benchmark matrix or an approved successor benchmark.

## Decision

Keep the rewritten ContextBuilder as the default implementation for VS Code context assembly and use the P01–P10 benchmark matrix as the standing regression comparison framework.

### Alternatives Considered

- **Keep the old broader strategy as default** — Rejected: The benchmark showed substantially higher estimated token cost with weaker relevance and weaker continuity.
- **Treat the rewrite as experimental only** — Rejected: The benchmark conclusion and third-take repeatability pass provide enough evidence to justify default adoption.
- **Record only a release note and no ADR** — Rejected: The decision affects future regression interpretation and benefits from durable guard rails.

## Consequences

- Clear long-term rationale for the rewritten context assembly design.
- Easier future regression evaluation using the same P01–P10 benchmark matrix.
- Stronger release justification for the current default implementation.
- Lower risk of reverting to broader context assembly without evidence.

**Risks accepted:**
- Benchmark artifacts are structured comparative captures rather than raw serialized telemetry.
- Broad architecture prompts may still require wider context than tightly scoped prompts.
- Future prompt or retrieval changes may require benchmark refresh.

## Guard Rails

> ⚠️ Do NOT replace the rewritten ContextBuilder default with a broader baseline strategy without rerunning the P01–P10 benchmark matrix or an approved successor benchmark.

> ⚠️ Do NOT describe these benchmark artifacts as raw telemetry.

> ⚠️ Revisit this decision if future benchmark reruns show sustained regression in relevance, continuity, or token efficiency.
