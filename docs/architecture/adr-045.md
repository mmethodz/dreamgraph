# ADR-045: Tension evidence must preserve semantic anchors as primary and treat line numbers as optional hints

**Date:** 2026-04-20T11:31:41.190Z  
**Status:** accepted  
**Decided by:** collaborative  
**Affected entities:** feature_architect_structured_webview, workflow_architect_agentic_tool_execution  

## Context

Long-lived tensions and cognitive evidence become brittle when preserved primarily as file line-number references. Source files evolve, causing line offsets to drift and weakening the durability and auditability of tension evidence across time, dream cycles, and repository changes.

**Constraints:**
- Tension evidence must remain durable across routine source edits and refactors.
- Cognitive artifacts need traceable provenance without depending on unstable positional coordinates.
- Existing code references and line ranges remain useful for short-term inspection and debugging, but must not be treated as canonical long-lived evidence.
- The evidence model should support future normalization, dream cycles, and cross-instance sharing without fragile source offsets.

## Decision

DreamGraph will preserve semantic evidence as the primary long-lived basis for tensions and cognitive insights. Canonical evidence should prefer semantic anchors such as entity names, feature IDs, workflow IDs, repository responsibilities, aggregate boundaries, save/sync behaviors, and relationship descriptions. File paths and line ranges may be stored only as optional cached hints for operator convenience and must not be required for the tension to remain meaningful or valid over time.

### Alternatives Considered

- **Continue storing line-number references as primary evidence** — Rejected: This makes long-lived tensions stale as files change and weakens durability across refactors.
- **Remove source-code location references entirely** — Rejected: Line hints and file references still help short-term inspection and debugging, so they remain useful as secondary evidence.
- **Store only raw code excerpts as canonical evidence** — Rejected: Raw excerpts drift too, are harder to normalize across instances, and are less queryable than semantic anchors.

## Consequences

- Tensions remain interpretable even after source files shift.
- Cross-cycle and cross-instance cognitive evidence becomes more durable.
- Operators can still navigate to file regions when hints are available, but evidence quality no longer depends on stable offsets.
- Future graph tooling can prioritize semantic linking and entity-based provenance over positional references.

**Risks accepted:**
- Existing historical tensions may still contain fragile line-based evidence and need gradual normalization.
- Some semantic anchors may be underspecified when the graph is sparse, requiring fallback hints until the graph is enriched.
- Tooling may need follow-up updates to ensure new tensions are recorded with semantic-first evidence.

## Guard Rails

> ⚠️ Do NOT treat file line numbers as the canonical long-lived identity or primary justification of a tension.

> ⚠️ Do NOT discard semantic entity/workflow/behavior evidence in favor of positional references when both are available.

> ⚠️ Do NOT require stable line offsets for a tension or cognitive insight to be considered valid over time.

> ⚠️ If line numbers are stored, treat them as optional cached navigation hints only.
