# ADR-080: Domain Boundaries for Instance Management

**Date:** 2026-04-25T11:55:43.164Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** instance_management, policy_management  

## Context

Clear separation of concerns is needed to manage different aspects of the application.

**Constraints:**
- Each domain must be independently manageable.

## Decision

Instance management is isolated from policy management.

### Alternatives Considered

- **Combine both domains** — Rejected: Increased complexity and tighter coupling.

## Consequences

- Easier maintenance and scalability.

**Risks accepted:**
- Potential for duplicated logic if not managed properly.

## Guard Rails

> ⚠️ Do NOT change domain boundaries without reviewing this ADR.
