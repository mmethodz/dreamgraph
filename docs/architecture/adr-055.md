# ADR-055: Use of Supabase for Transactional Data Storage

**Date:** 2026-04-25T11:03:26.848Z  
**Status:** accepted  
**Decided by:** system  
**Affected entities:** data_model_feature, data_model_workflow  

## Context

Need a reliable and scalable solution for managing transactional data.

**Constraints:**
- All transactional data must be stored in Supabase.

## Decision

Supabase for transactional data storage.

### Alternatives Considered

- **PostgreSQL** — Rejected: Requires additional management overhead.
- **MongoDB** — Rejected: Not suitable for relational data.

## Consequences

- Simplified data management and scalability.

**Risks accepted:**
- Dependency on Supabase service availability.

## Guard Rails

> ⚠️ Do NOT switch data storage solutions without reviewing this ADR.
